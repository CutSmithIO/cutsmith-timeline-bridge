# Relink Guide — cutsmith2

Generated by `cutsmith collect`. Keep this file with the package so any editor who opens it knows where to find the media.


## Importing into Premiere

1. `File → Import…`, select `cutsmith2.xml`.
   **Do not** use `Open Project`.
2. Premiere creates a Sequence and Project panel source items.
3. All assets should link automatically — no manual relink step needed.


## If clips appear as Offline

Right-click any offline clip in the Project panel → **Link Media…**
Navigate to:

    /Users/mac/cutsmith/cutsmith-timeline-bridge/out_collect/cutsmith2/media

Premiere will auto-link remaining offline clips by name from the same folder (this works because every clip carries a `<masterclipid>` back-reference to its source item).


## Media folder layout

```
media/
├── video/      ← user video clips
├── audio/      ← user audio / voiceover
├── images/     ← image overlays and thumbnails
├── music/      ← CapCut music library tracks *
├── sfx/        ← CapCut SFX *
└── stickers/   ← sticker assets (rare)
```

\* **Third-party asset licensing**: music, SFX, and sticker files copied from the CapCut cache may carry licensing terms from CapCut or third-party rights holders. Copying them into a portable package does not transfer usage rights. Verify distribution rights before publishing content that includes CapCut library assets.


## Assets NOT included

The following are **not portable** outside CapCut and are absent from `media/`:

- CapCut effects, video effects, plugin effects
- CapCut transitions
- CapCut filters and HSL adjustments
- CapCut fonts (install the font on the destination machine manually)

Rebuild these using Premiere's native effects or third-party plugins. See `cutsmith2.report.md` for the full list of dropped items.


## Speed-changed clips

CutSmith emits an explicit FCP7 `timeremap` filter so Premiere reconstructs Speed/Duration on import (e.g. 200%, 49.91%). No manual Speed/Duration adjustment is needed after import.

Variable-speed curves (`speed_curve`) are report-only — those clips play at 1.0× in Premiere. Reconstruct via Effect Controls → Time Remapping → Velocity if needed.


### Known edge case — trim boundary on speed-changed clips

When a speed-changed clip (e.g. 2×) uses the entire source file, Premiere may show **wavy trim handles** if you try to extend it beyond its current boundaries. This is expected behaviour, not a CutSmith bug:

- The exported timeline slot duration is correct.
- The wavy lines mean the source material has no additional frames beyond the file boundary.
- **To recover frames:** use a longer source take in CapCut and re-export, or manually relink the clip to a longer file in Premiere.
