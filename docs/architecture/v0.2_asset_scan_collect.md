# CutSmith v0.2+ Architecture — Asset Scan, Collect, Subtitle Export

_Status: design-only — no code written yet. Implementation gated on 0519 / 0519V validation results._

---

## 0. Core product direction

From "XML converter" to "CapCut Project → Professional Workflow Bridge":

```
v0.1  CapCut draft → FCP7 XML

v0.2+ CapCut draft → { XML, SRT, AssetManifest, CollectedMedia }
```

**Unchanged constraint**: we do not reconstruct CapCut's render effects. The mission is to safely move rough cuts and referenced media into professional workflows — extract, collect, relink — not to replicate CapCut inside Premiere.

## 1. IR / manifest schema

### 1.1 Extend `MediaAsset` in `cutsmith/ir/__init__.py`

Two new fields, backward-compatible (have defaults):

```python
class AssetClass(str, Enum):
    USER_VIDEO      = "user_video"     # user's own video file
    USER_AUDIO      = "user_audio"     # user's own audio (not a music library track)
    USER_IMAGE      = "user_image"     # user's own image
    CAPCUT_MUSIC    = "capcut_music"   # CapCut music library track (cached)
    CAPCUT_SFX      = "capcut_sfx"     # CapCut sound effect (cached)
    CAPCUT_STICKER  = "capcut_sticker" # CapCut sticker / overlay
    CAPCUT_FONT     = "capcut_font"    # CapCut-downloaded font
    CAPCUT_EFFECT   = "capcut_effect"  # CapCut effect or filter asset
    UNKNOWN         = "unknown"

@dataclass
class MediaAsset:
    ...  # all existing fields unchanged
    asset_class: AssetClass = AssetClass.UNKNOWN   # filled by scanner, not reader
    is_cached: bool = False                         # True when path is inside CapCut cache dir
```

Classification is the scanner's responsibility; the reader stays focused on producing the IR.

### 1.2 New IR types for subtitles — `SubtitleCue`, `SubtitleTrack`

Added to `cutsmith/ir/__init__.py` in v0.2:

```python
@dataclass
class SubtitleCue:
    cue_id: str
    start_us: int
    end_us: int
    text: str                       # plain text, no styling
    font_name: str | None = None    # if detectable
    is_auto_caption: bool = False   # recognize_type != 0 → ASR-generated

@dataclass
class SubtitleTrack:
    track_id: str
    cues: list[SubtitleCue]
    cue_count: int                  # = len(cues)
    likely_caption_track: bool      # heuristic: cue_count > 5
```

`Timeline` gains:

```python
@dataclass
class Timeline:
    ...  # existing fields unchanged
    subtitle_tracks: list[SubtitleTrack] = field(default_factory=list)  # v0.2
```

### 1.3 Standalone `AssetManifest` — output schema for `scan-assets`

This is a distinct output artifact, not part of the IR. It is richer than the IR's `MediaAsset` list and is designed to be GUI-friendly (flat, UTF-8, no binary).

```python
@dataclass
class ManifestEntry:
    asset_id: str
    name: str
    asset_class: AssetClass
    original_path: str | None
    resolved_path: str | None
    is_cached: bool
    is_online: bool               # resolved_path exists and is a file
    file_size_bytes: int | None   # None if offline
    duration_us: int | None
    used_in_tracks: list[str]     # track names
    clip_count: int
    collect_relative_path: str | None = None  # filled by collector

@dataclass
class AssetManifest:
    schema_version: int = 1       # bump when schema changes
    project_name: str = ""
    source_draft: str = ""
    app_version: int = 0
    duration_us: int = 0
    # categorised entries
    videos:   list[ManifestEntry] = field(default_factory=list)
    audios:   list[ManifestEntry] = field(default_factory=list)
    music:    list[ManifestEntry] = field(default_factory=list)
    sfx:      list[ManifestEntry] = field(default_factory=list)
    images:   list[ManifestEntry] = field(default_factory=list)
    stickers: list[ManifestEntry] = field(default_factory=list)
    effects:  list[ManifestEntry] = field(default_factory=list)  # report-only
    fonts:    list[ManifestEntry] = field(default_factory=list)  # report-only
    offline:  list[ManifestEntry] = field(default_factory=list)  # all is_online=False
    # stats
    total_assets: int = 0
    online_count: int = 0
    offline_count: int = 0
    cached_count: int = 0
    total_online_size_bytes: int = 0
```

JSON output: every `ManifestEntry` serialises as a flat dict — no nesting. `schema_version` allows GUI consumers to gate on breaking changes without parsing the full body.

## 2. `scan-assets` architecture

### 2.1 Module layout

```
cutsmith/
  scanner/
    __init__.py     # scan_assets(draft_path, search_roots) → AssetManifest
    classify.py     # AssetClass determination
    cache.py        # CapCut cache path heuristics
    probe.py        # file size probe (no ffprobe dependency)
```

### 2.2 Execution order inside `scan_assets()`

```
1. detect(draft_path)              validate format, reject encrypted
2. read_draft(draft_path)          → Timeline (existing reader)
3. resolve_media_paths(timeline)   populate resolved_path
4. _read_extra_materials(draft)    read categories the reader intentionally skips:
                                   stickers, effects (type=filter included), fonts,
                                   text_templates, texts
5. classify_assets(timeline)       fill asset_class + is_cached on each MediaAsset
6. build_usage_map(timeline)       asset_id → [track_name, clip_count]
7. probe_sizes(timeline)           fill file_size_bytes for online assets
8. assemble_manifest()             → AssetManifest
```

Step 4 reads only what the reader deliberately skips. It does **not** re-read the whole draft — it loads the same JSON once and slices the categories the reader ignores.

### 2.3 Asset classification logic (`classify.py`)

Precedence order:

```python
def classify(asset: MediaAsset, raw_mat: dict | None, cache_roots: set[Path]) -> AssetClass:
    path = asset.original_path or ""

    # 1. Path is inside a known CapCut cache directory
    if _is_cache_path(path, cache_roots):
        mat_type = (raw_mat or {}).get("type", "")
        if mat_type == "music":
            return AssetClass.CAPCUT_MUSIC
        if mat_type in ("sound", "sfx"):
            return AssetClass.CAPCUT_SFX
        return AssetClass.CAPCUT_EFFECT

    # 2. Path is inside the CapCut / JianyingPro app bundle (fonts, built-in assets)
    if "/CapCut.app/" in path or "/JianyingPro.app/" in path:
        return AssetClass.CAPCUT_FONT

    # 3. Material type field distinguishes music from user audio
    mat_type = (raw_mat or {}).get("type", "")
    if mat_type == "music":
        return AssetClass.CAPCUT_MUSIC
    if mat_type in ("sound", "sfx"):
        return AssetClass.CAPCUT_SFX
    if mat_type == "video_original_sound":
        return AssetClass.USER_AUDIO   # embedded audio extracted from a video clip

    # 4. Fall back to media_kind
    if asset.media_kind == MediaKind.VIDEO:
        return AssetClass.USER_VIDEO
    if asset.media_kind == MediaKind.AUDIO:
        return AssetClass.USER_AUDIO
    if asset.media_kind == MediaKind.IMAGE:
        return AssetClass.USER_IMAGE

    return AssetClass.UNKNOWN
```

**Music vs SFX refinement**: CapCut cache paths typically have `music/` and `sfx/` subdirectories — classification can be further refined by `path.split("/")` without relying solely on `type`.

### 2.4 CLI interface

```
python3 -m cutsmith scan-assets <project> [-o OUT_DIR] [--json] [-s ROOT...]
```

- No `-o`: print human-readable summary to stdout; `--json` switches to JSON.
- With `-o`: write `{name}.manifest.json` + `{name}.scan.md`.

## 3. Collect / relink strategy

### 3.1 Output directory structure

```
<output>/
  <name>.xml               ← relinkable FCP7 XML (paths rewritten to collected locations)
  <name>.report.md
  <name>.manifest.json     ← manifest with collect_relative_path filled
  <name>.offline.md        ← only generated when offline assets exist
  media/
    video/
    audio/
    music/
    sfx/
    images/
    stickers/              ← copied only; Premiere can't render CapCut stickers,
                              but the file is preserved for reference
```

### 3.2 Copy pipeline (`cutsmith/collector/__init__.py`)

```
collect(project, output_dir, search_roots):
  1. scan_assets(project, search_roots) → manifest
  2. for each online entry in manifest:
       dest = output_dir / "media" / _subdir(entry.asset_class) / entry.name
       if collision (same name, different asset_id):
           dest = stem + "_{asset_id[:8]}" + suffix
       shutil.copy2(src, dest)
       entry.collect_relative_path = str(dest.relative_to(output_dir))
  3. rewrite_xml(timeline, manifest, output_dir)   # path override map
  4. write_collect_report(manifest, output_dir)
```

### 3.3 XML path rewriting

`fcp7_xml.py`'s `_pathurl()` currently uses `asset.resolved_path` directly. Collect mode passes a `path_override: dict[asset_id, str]` to the writer. The writer prioritises the override. The writer has no knowledge of the collect pipeline — it only accepts a map.

```python
def write_xml(timeline, output_path, path_override=None):
    # path_override: {asset_id: absolute_collected_path}
    ...
```

### 3.4 Filename collision handling

When two assets share a basename but different `asset_id` (same-named file from different directories):

```
media/video/
  shoot_day1.mp4             ← first occurrence, no suffix
  shoot_day1_3f2a9b1c.mp4   ← collision: append first 8 chars of asset_id
```

Full UUIDs are avoided to keep paths readable.

## 4. Cache asset handling strategy

### 4.1 Known CapCut cache roots on macOS

```python
KNOWN_CACHE_ROOTS = [
    "~/Library/Containers/com.lemon.lvoverseas/Data/Movies/CapCut/User Data/Cache",
    "~/Library/Containers/com.lemon.lvoverseas/Data/Library/Caches",
    "~/Library/Group Containers/group.com.lemon.lvoverseas",
    "~/Movies/CapCut/User Data/Cache",
    "~/Movies/JianyingPro/User Data/Cache",
]
```

`cache.py` expands these with `Path.expanduser()` and checks `Path.is_relative_to()`. Additional roots can be auto-detected at runtime (scan for the `com.lemon.lvoverseas` container).

### 4.2 Decision table

| State | Action |
|---|---|
| Cache file present on disk | Copy to `media/music/` or `media/sfx/`. Mark `is_cached=True` in manifest. |
| Cache file evicted by CapCut | Mark `is_online=False`. Add to offline report with: "CapCut cache asset — re-download in CapCut, then re-run `cutsmith collect`." |
| Path inside CapCut app bundle (fonts) | Do **not** copy — app distribution licence applies. List font name in scan report only. |

### 4.3 Licensing note in collect report

CapCut music library tracks are included in the collected output for archival and workflow continuity. Their licensing may be restricted to CapCut / TikTok platforms. Verify licensing terms before distributing the collected project.

|---|---|---|

### 5.2 Offline clips in the XML

Offline assets are not omitted from the XML — they get a clearly-named placeholder pathurl (current v0.1 behaviour). Premiere opens the XML with those clips marked offline and shows "Link Media" when the user clicks them. The placeholder name matches the original filename, giving Premiere's relink dialog a meaningful search target.

## 7. Subtitle extraction — two distinct patterns

Field research on real CapCut Desktop 167.0.0 drafts reveals two structurally different text track patterns. The subtitle extractor must handle both.

### Pattern A — Simple text (e.g. cutsmith2)

```
text_track_segment.material_id → materials.texts[].id
texts[].content  (JSON string) → .text  (plain string)
```

Detection: `materials.texts[id].content` is a JSON string with a top-level `"text"` key.

### Pattern B — Animated subtitle template (e.g. 0519V)

```
text_track_segment.material_id
    → materials.text_templates[].id
      .text_info_resources[].text_material_id
          → materials.texts[].id
            .content (JSON) → .text
```

Detection: `materials.text_templates[id].type == "text_template_subtitle"`.

### Extraction algorithm (v0.2)

```python
def extract_text(segment, text_mats_by_id, text_tmpl_by_id):
    mid = segment["material_id"]

    # Pattern A: direct texts reference
    if mid in text_mats_by_id:
        return _parse_text_content(text_mats_by_id[mid])

    # Pattern B: text_template wrapper
    if mid in text_tmpl_by_id:
        tmpl = text_tmpl_by_id[mid]
        for res in tmpl.get("text_info_resources", []):
            text_mid = res.get("text_material_id", "")
            if text_mid in text_mats_by_id:
                return _parse_text_content(text_mats_by_id[text_mid])

    return ""  # unknown pattern — safe fallback

def _parse_text_content(mat: dict) -> str:
    try:
        return json.loads(mat.get("content", "{}")).get("text", "")
    except (json.JSONDecodeError, AttributeError):
        return mat.get("recognize_text", "")  # ASR fallback
```

## Key design decisions

1. **Scanner does not merge into reader.** Reader contract: CapCut schema → IR. Scanner contract: IR + raw materials → AssetManifest. Separating them keeps the reader testable in isolation.

2. **`AssetManifest` is independent of the Timeline IR.** IR is the reader→writer internal interface. Manifest is the user-facing output interface. Each has its own versioning.

3. **Collect depends on scan; neither depends on inspect.** `inspect` is a raw-schema debug tool. `collect` only needs the manifest.

4. **GUI deferred until CLI + manifest schema are stable.** All JSON outputs are designed to be GUI-consumable today (flat, UTF-8, no binary). PySide6 wrapping is additive, not a structural change.

5. **v0.2/v0.3 ordering depends on Premiere speed test.** If Premiere does not auto-interpret the implicit speed (end-start ≠ out-in), an explicit Time Remap `<filter>` rises above scan-assets in priority.
